## 1. 逻辑缺陷修复

### 1.1 `Settings.cpp` — `notCycleOrHold()` 条件错误

**问题：** 使用 `&&` 导致条件恒为 `false`，Cycle/Hold 状态下本应被拦截的 Web/`[ESP...]` 命令可能仍可执行。

**修改：**

```cpp
// 修复前
return sys.state == State::Cycle && sys.state == State::Hold;

// 修复后
return sys.state == State::Cycle || sys.state == State::Hold;
```

**影响：** 绑定 `notCycleOrHold` 检查器的命令在运行或暂停期间将正确返回 `Error::IdleError`。

---

### 1.2 `MotionControl.cpp` — 探针 `"Found"` 误报

**问题：** 在探针运动开始前即打印 `"Found"`，误导操作者。

**修改：**

- 删除 `limitsCheckSoft(target)` 之前的 `"Found"` 日志
- 在 `sys.probe_succeeded = true` 成功分支内打印 `"Found"`

---

## 2. 性能优化

### 2.1 `TelnetServer.cpp` — Telnet 输入日志

**问题：** `push(uint8_t data)` 对每个字符调用 `log_i`，高流量 Telnet 输入会拖慢主循环。

**修改：** 移除 `push()` 中的逐字符及 buffer size 的 `log_i` 调用。

---

## 3. WebUI 与 SPIFFS 任务执行

### 3.1 `WebSettings.cpp` — `runLocalFile`（ESP700）

**问题：** `runLocalFile` 使用硬编码 `char line[256]` 与 `getBytes(line, 255)`，与 Grbl 协议层 `LINE_BUFFER_SIZE` 定义不一致。

**修改：**

```cpp
char line[LINE_BUFFER_SIZE];
currentline.getBytes(line, LINE_BUFFER_SIZE - 1);
```

**说明：** `LINE_BUFFER_SIZE` 在 `Protocol.h` 中默认为 256，行为与 Grbl 主协议一致，并保留 NUL 终止空间。

---

## 4. 定制纸路模块（`Custom/paper_system.cpp`）

### 4.1 非阻塞等待与运动互锁

**问题：** 原实现使用 `delay()` 阻塞主循环；审查期间缺少与 Grbl 状态机的显式互锁。

**修改：**

- 新增 `paper_motion_allowed()`：在 `Alarm` / `Cycle` / `Hold` / `Homing` 或 `sys.abort` 时禁止纸路动作
- 新增 `paper_blocking_wait()`：以 `delay_msec()` 替代 `delay()`，可在等待期间响应 abort

**注意：** 真机仍需验证进纸/退纸与 XYZ 联动及 Hold/Alarm 行为。

1. **默认口令：** `admin` / `user` — **首次部署务必修改**

3. **Telnet 已关闭：** 远程 G-code 请用 USB/BT/HTTP
4. 设备应部署在受信任局域网
