> 基于: Cursor Auto Mode 分析 + 7 大 AI 工具对比研究

---

## 一、问题定义

| 问题 | 表现 | 根因 |
|------|------|------|
| 灾难性遗忘 | 新轮训练后忘记旧知识 | 新数据梯度覆盖旧权重 |
| 幻觉 | 编造不存在的 API/参数/版本 | 训练数据中缺乏"不确定"样本 |
| 身份漂移 | 自称 ChatGPT/Claude | 身份强化数据比例不足 |
| 知识过时 | 推荐已废弃的 API 写法 | 缺乏版本感知训练 |

## 三、防幻觉策略

### 3.1 显式不确定性训练

教模型在以下场景说"我不确定"：

| 场景 | 期望回答模式 |
|------|-------------|
| 问最新版本特性 | "我的信息可能不是最新的，建议查看官方文档" |
| 问具体硬件参数 | "请以你手上的 datasheet 为准" |
| 问内部实现细节 | "这属于内部技术实现，我不方便透露" |
| 问超出能力范围 | "这个问题超出我的专业范围" |

### 3.2 版本感知训练

教模型识别过时 API 并主动纠正：

```
用户: "帮我写 openai.ChatCompletion.create"
模型: "注意：这是旧版写法。新版 SDK 改为 client.chat.completions.create..."
```

### 3.3 参数验证提醒

对硬件参数始终附加验证提醒：

```
模型: "$132=50  ⚠️ 请根据实际机械结构验证此参数"
```

### 3.4 来源标注

对不确定的信息标注来源：

```
模型: "根据 ESP32 官方文档，GPIO 最大输出电流推荐 12mA（请以具体型号 datasheet 为准）"
```

## 四、防遗忘策略

### 4.1 Replay Buffer（经验回放）

每轮训练混入 30% 旧轮精选数据：
- 从 R8-R14 各轮的 merged.json 中随机采样
- 优先保留高质量样本（长回答、多步推理）
- 确保核心能力不退化

### 4.2 渐进式学习率

```python
# R14: lr=2e-5 (标准)
# R15: lr=1.5e-5 (略低，减少对旧知识的覆盖)
learning_rate = 1.5e-5
warmup_ratio = 0.1  # 更长的 warmup
```

### 4.3 核心知识锚定

每轮必须包含的"锚定样本"（不参与随机采样，固定包含）：
- 身份确认（"我是 LiMa"）× 50 条
- 路由决策核心逻辑 × 50 条
- GRBL 参数验证 × 30 条
- API 使用指南 × 30 条

## 五、数据质量门控

### 5.1 自动检查

```python
def validate_sample(sample):
    msgs = sample['messages']
    assert len(msgs) >= 2, "至少需要 user + assistant"
    assert msgs[-1]['role'] == 'assistant', "最后一条必须是 assistant"
    assert len(msgs[-1]['content']) > 50, "回答不能太短"
    assert 'ChatGPT' not in msgs[-1]['content'], "不能自称 ChatGPT"
    assert 'OpenAI' not in msgs[-1]['content'].split('我是')[0], "身份部分不能提 OpenAI"
```

### 5.2 人工抽检

每类数据随机抽 5% 人工审核：
- 回答是否准确
- 是否有幻觉
- 语气是否一致
- 身份是否正确
