## 关键发现

### 🟡 P1: VPS TTFB 仍然偏高（3.5s）

虽然比本地快，但依然超目标 73%。

**可能原因**:
1. **后端选择问题**
   - 当前后端可能不是最快的（需要测试 groq/cerebras）
   - 路由决策可能偏向质量而非速度

2. **冷启动或负载**
   - 第一次请求可能触发模型加载
   - VPS 可能有其他请求占用资源

3. **非流式响应**
   - 只有 1 个 chunk 说明后端可能不支持真流式
   - 或 LiMa 的流式转换有问题

**优化方向**:
- [ ] 测试 `groq_llama3370b` 后端（已知最快）
- [ ] 增加路由日志查看决策耗时
- [ ] 检查为什么是非流式响应

---

### 🔴 P0: 工具调用完全不可用

本地和 VPS 都失败，这是核心功能问题。

**紧急排查**:
