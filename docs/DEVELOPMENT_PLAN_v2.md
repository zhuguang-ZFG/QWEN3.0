## 三、套餐与计费配置（优先级：P1）

### 目标
在 one-api 中配置套餐体系

### 套餐设计

| 套餐 | 月费 | 日调用 | 单次token | 特权 |
|------|------|--------|-----------|------|
| 免费 | ¥0 | 100次 | 2K | 基础路由 |
| 基础 | ¥29 | 3000次 | 8K | 全模型+优先 |
| 专业 | ¥99 | 无限 | 32K | 全模型+专属 |
| 企业 | 定制 | 定制 | 定制 | 私有部署 |

### 实现方式
- one-api 用户组 + 配额机制
- 免费用户：注册即得 100次/天 额度
- 付费用户：手动充值或对接支付（后期）

### 验收标准
- [ ] 新注册用户自动获得免费额度
- [ ] 超出额度返回 429 + 友好提示
- [ ] 管理后台可查看用量统计

## 四、监控与告警（优先级：P2）

### 目标
确保服务稳定性，异常时及时通知

### 4.1 frp 隧道监控
- 方案：cron 定时 curl 本地端口，失败时发通知
- 通知渠道：企业微信/钉钉 webhook 或邮件
- 频率：每 5 分钟检测一次

### 4.2 服务健康检查
- one-api 健康端点：`/api/status`
- LiMa 本地服务：`http://127.0.0.1:8080/health`
- 云端定时检测两个端点

### 4.3 日志收集
- one-api 日志：systemd journal
- Nginx 访问日志：`/var/log/nginx/access.log`
- 本地 LiMa 日志：`D:/GIT/logs/`

### 验收标准
- [ ] frp 断线 5 分钟内收到通知
- [ ] 有日志可追溯过去 7 天的请求

## 七、技术决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 模型名 | LiMa (力码) | 品牌关联强，谐音"立马"，国际化友好 |
| 版本号 | 1.3 | R14 = 第14轮训练，对外版本 1.3 |
| 官网框架 | Next.js 16 + static export | 已部署，SEO友好，CDN可缓存 |
| 监控方案 | cron + webhook | 轻量，不需要额外基础设施 |
| 支付 | 暂不接入 | 先手动充值，验证PMF后再接支付 |
