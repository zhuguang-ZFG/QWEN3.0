## 归档规则

1. **仅文档**：代码文件（`.py`/`.ts` 等）不放入此目录，应直接删除（git 历史可恢复）。
2. **按主题/时间分子目录**：如 `strategic-plans-2026-06/`、`superpowers-2026-05/`。
3. **不修改内容**：归档后只读，不更新；如需纠正，在活跃文档中注明。
4. **定期审查**：超过 6 个月且无引用的归档文档可考虑删除。
