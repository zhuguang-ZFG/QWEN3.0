## 🔍 深入分析

### 1. 延迟改善趋势
- **第 1-2 轮**: 轻微改善（~4%）- 会话缓存初始化
- **第 3-5 轮**: 显著改善（12-17%）- 会话缓存完全生效

### 2. 后端一致性
- **基线**: 1 个独立后端（每次请求可能重新选择）
- **优化**: 1 个独立后端（会话缓存锁定后端）
- **结论**: 会话缓存有效维持后端亲和性

### 3. Token 统计
- **状态**: 未获取到 token 使用数据
- **原因**: 后端响应中 `usage` 字段为 0
- **影响**: 无法验证 token 节省，但不影响延迟改善验证
