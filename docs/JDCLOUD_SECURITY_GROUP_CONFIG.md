# 京东云安全组配置指南

## 问题诊断结果

**连接失败原因**: 京东云安全组未放行阿里云 IP

- ✅ Redis 服务正常（监听 0.0.0.0:6379）
- ✅ UFW 防火墙规则正确
- ❌ **云平台安全组阻止了连接**

---

## 解决步骤

### 1. 登录京东云控制台

访问: https://console.jdcloud.com

### 2. 找到云主机安全组

```
路径:
  弹性计算 → 云主机 → 实例列表 
  → 找到 117.72.118.95 
  → 点击实例名称
  → 安全组标签页
```

### 3. 添加入站规则

点击"配置规则"或"编辑规则"，添加：

```
类型: 自定义 TCP
协议: TCP
端口范围: 6379
源地址: 47.112.162.80/32
描述: LiMa Redis from Aliyun
操作: 允许
```

### 4. 保存并验证

保存后等待 1-2 分钟，然后测试：

```bash
# 从阿里云 VPS 测试
ssh root@47.112.162.80

# 测试 Ping
ping -c 3 117.72.118.95

# 测试 Redis
redis-cli -h 117.72.118.95 -p 6379 -a 'reu/0E4Y3k+5yyaFqFbL6V1uw6wfs0UXfZk145xpp/Q=' ping
```

## 备选方案

如果京东云安全组配置复杂或无法配置，有以下备选方案：

### 方案 A: 放弃京东云 Redis，改用阿里云本地 Redis

```bash
# 直接在阿里云 VPS 安装 Redis
ssh root@47.112.162.80
apt install redis-server
# 配置后直接使用 127.0.0.1:6379
```

**优点**:
- 无网络延迟
- 无需跨云通信
- 配置简单

**缺点**:
- 占用阿里云 VPS 资源
- 京东云服务器闲置

### 方案 B: 使用内网穿透（frp/ngrok）

配置 frp 隧道穿透云安全组限制。

**不推荐**（复杂且不稳定）

### 方案 C: 使用京东云内网服务

如果两台 VPS 在同一区域，可以配置内网互通。

## 推荐行动
