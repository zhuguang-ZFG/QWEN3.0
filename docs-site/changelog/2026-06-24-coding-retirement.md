# 2026-06-24 编码能力退役

**类型**：能力调整
**影响范围**：路由分类器、聊天入口、控制台

## 变更说明

经过对使用数据与维护成本的评估，LiMa 决定退役面向 IDE/代码编辑场景的编码辅助能力，将精力集中在聊天、图像生成与设备控制三大核心场景。

## 具体调整

- 移除代码编辑、IDE 插件相关路由与入口。
- `routing_classifier` 不再将请求导向 `ide`/`code` 场景。
- 相关测试、文档与示例逐步归档到 `docs/archive/`。

## 用户影响

- 通过 OpenAI/Anthropic 兼容 API 发送的通用对话请求不受影响。
- 需要代码生成能力的用户仍可调用通用聊天模型，模型会按通用对话返回代码片段。
