# Product

## Register

brand

## Users

**主要受众**：智能硬件创作者与小工作室运营者——购买或关注 AI 绘图机、AI 写字机、2D 数字人设备的个人用户与小团队。他们在选购前浏览官网了解产品能力、价格、技术原理，决策依据是「这个平台能不能让我的设备真正创作出好作品」。

**次要受众**：开发者与集成商——通过 OpenAI 兼容 API 接入 LiMa 后端，将 170+ AI 模型路由能力嵌入自己的应用。他们关注 API 文档、playground、技术架构透明度。

**使用场景**：移动端与桌面端浏览，首次访问 30 秒内需要理解「LiMa 是什么、能做什么、跟我有什么关系」。

## Product Purpose

LiMa 量子星云系统是 AI 智能设备的统一云端服务。它连接 170+ AI 后端（Groq、NVIDIA、OpenRouter、DeepSeek、Cloudflare 等），通过量子路由、多模态坍缩与设备纠缠协同，把自然语言指令转化为绘图机笔触、写字机笔画、数字人表情——让 AI 创意走出屏幕、变成纸上的真实作品。

官网存在的目的：让访客在 60 秒内理解 LiMa 的核心价值（「一句话驱动真实创作」），建立技术信任感，并引导至在线体验或 API 接入。

## Brand Personality

**技术诗意 / 克制自信 / 创造落地**

- 技术诗意：用「量子」「星云」「坍缩」「纠缠」等物理隐喻描述 AI 路由，但不浮夸——每个隐喻都对应真实技术机制。
- 克制自信：不喊口号，用 170+ 后端、真实设备、实测数据说话。深色界面传递专业感，cyan 高光传递前沿感。
- 创造落地：强调「从屏幕到纸张」「从代码到笔触」的物理转化，这是 LiMa 区别于纯软件 AI 平台的核心差异。

**情感目标**：访客离开时记住的是「这个平台能让 AI 真正画出/写出东西」，而非「又一个 AI API 聚合器」。

## Anti-references

- **不要像通用 AI API 聚合器官网**（OpenRouter、Together AI 风格）：纯功能列表 + 模型表格 + 紫蓝渐变。LiMa 的差异是「设备」，官网必须让设备可见。
- **不要像企业 SaaS 蓝白官网**：浅色背景 + 蓝色按钮 + 三栏 feature 卡 + 客户 logo 墙。这种风格稀释了技术深度感。
- **不要 AI slop**：Inter 字体铺满、紫蓝渐变按钮、卡片套卡片、灰字压彩底、圆角图标方块、emoji 当 icon。impeccable 44 条检测规则全部生效。
- **不要过度装饰的科技风**：满屏粒子动画、矩阵雨、闪烁霓虹。LiMa 的科幻感来自克制的几何线条与星云意象，不是特效堆砌。

## Design Principles

1. **设备即证据**：每个产品介绍都必须配真实设备图片或运行可视化，而非抽象图标。绘图机在画、写字机在写、数字人在动——这就是说服力。
2. **隐喻可追溯**：每个量子/星云隐喻都锚定一个可解释的技术机制。路由=量子坍缩（多后端竞速选最优）、多模态=纠缠协同、设备调度=观测者效应。不堆砌不可解释的术语。
3. **深色为底，光点为引**：深色背景传递技术深度，cyan/紫的高光只用于引导视线与强调交互，不做装饰性铺满。留白即星云。
4. **一行胜千言**：核心价值用一句话说清（「把自然语言坍缩为真实创作」），细节交给滚动探索。首屏不堆 feature 列表。
5. **开发者友好但不喧宾夺主**：API/playground 是次要受众入口，放在导航次级，不抢占设备创作者的首屏注意力。

## Accessibility & Inclusion

- **WCAG 2.1 AA**：深色主题下所有文本对比度 ≥ 4.5:1（slate-200 on #07070f 已满足，需持续检测新增内容）。
- **键盘导航**：SkipLink 已实现，所有交互元素键盘可达，focus-visible 样式已定义（cyan outline）。
- **减少动效**：`prefers-reduced-motion` 已在 globals.css 实现，Reveal 组件需遵守。
- **色彩无障碍**：cyan accent (#06b6d4) 对深色背景对比度充足；避免仅用颜色传达信息（状态需配图标或文字）。
- **中英双语**：zh-CN 为主，en/ 为完整镜像，不遗漏页面。
